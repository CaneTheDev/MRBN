# Contributing to MRBN

Thank you for your interest in contributing to MRBN! Like Bitcoin, MRBN is an open-source project that welcomes contributions from everyone.

## Philosophy

MRBN follows the same open-source principles as Bitcoin:
- Anyone can read the code
- Anyone can modify the code
- Anyone can redistribute the code
- The community drives development

## Ways to Contribute

### 1. Code Contributions
- Implement core protocol features
- Develop client applications
- Improve performance and security
- Write tests and documentation

### 2. Documentation
- Improve technical documentation
- Write tutorials and guides
- Translate documentation
- Create educational content

### 3. Research
- Analyze security models
- Propose protocol improvements
- Conduct performance testing
- Review cryptographic implementations

### 4. Community
- Help others in community channels
- Report bugs and issues
- Provide feedback on proposals
- Participate in discussions

## Development Process

### Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Write or update tests
5. Submit a pull request

### Code Standards

- Write clear, documented code
- Follow Rust best practices (for core implementation)
- Include tests for new features
- Maintain backward compatibility when possible

### Commit Messages

Use clear, descriptive commit messages:
```
Add VRF committee selection algorithm

Implements the Verifiable Random Function for probabilistic
committee selection as described in section 4.2 of the whitepaper.
```

## Pull Request Process

1. Ensure your code builds and passes all tests
2. Update documentation as needed
3. Describe your changes clearly in the PR
4. Be responsive to feedback and questions
5. Maintain a respectful, collaborative tone

## Security Issues

If you discover a security vulnerability:
- **DO NOT** open a public issue
- Email security details privately (contact info TBD)
- Allow time for the issue to be addressed before public disclosure

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on technical merit
- Accept constructive criticism gracefully
- Prioritize the project's long-term health

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or inflammatory comments
- Personal attacks
- Publishing others' private information
- Other conduct inappropriate in a professional setting
