# MRBN Testnet

**Status**: In Development

## Overview

The MRBN Testnet is the first working implementation of the MRBN protocol. It serves as a testing ground for:
- Core protocol validation
- Network stability testing
- Performance benchmarking
- Security testing
- Community feedback

## Testnet Goals

1. Prove the micro-resource validation model works
2. Test VRF-based committee selection at scale
3. Validate economic incentives and gas distribution
4. Identify and fix bugs before mainnet
5. Build community and gather feedback

## Testnet vs Mainnet

| Feature | Testnet | Mainnet |
|---------|---------|---------|
| Kain Value | No real value | Real economic value |
| Network Reset | Can be reset | Permanent |
| Purpose | Testing & development | Production use |
| Participation | Open to all testers | Open to all |

## Testnet Phases

### Phase 1: Internal Testing (Current)
- Core developers only
- 10-50 nodes
- Frequent resets expected
- Focus: Basic functionality

### Phase 2: Closed Beta
- Invited community members
- 100-500 nodes
- Weekly resets
- Focus: Stability and performance

### Phase 3: Public Testnet
- Open to everyone
- 1000+ nodes
- Monthly resets (if needed)
- Focus: Real-world conditions

### Phase 4: Mainnet Preparation
- Final testing
- Security audits
- No resets unless critical
- Focus: Production readiness
